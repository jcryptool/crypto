Diskussion guter und optimaler Strategien für das Zahlenhai-Spiel

In diesem Kapitel wird das Spiel Zahlenhai auf mathematischer und strategischer Ebene erörtert.

1. Definitionen

Zahlenmenge:
Das Zahlenhai-Spiel wird mit einer bestimmten Anzahl von Zahlen gespielt. Diese Menge wird Zahlenmenge genannt. Für die weitere Diskussion wird, wenn nicht ausdrücklich erwähnt, angenommen, dass die Zahlenmenge identisch ist mit {1,...,N}, wobei N ein Element von |N ist.

Pfad oder Zugfolge:
Eine Pfad- oder Zugfolge ist eine Folge von Zahlen aus der Zahlenmenge, die das Spiel für ein gegebenes N gemäß den Regeln löst. Die Zugfolge enthält nur die Zahlen des Spielers.

So sind zum Beispiel (19,15,14,20,16,12,18)(20) oder (19,10,15,20,12,14,18,16)(20) mögliche Zugfolgen für N=20.

Im Allgemeinen ist eine Zugfolge für N definiert als (a(m))(N) = (n1, n2,...,nm)(N), mit m, N ist Element von |N, N>=2, m<=N-1, i,j ist Element von {1,...,m}, ni ist Element der Zahlenmenge, ni != nj für i != j.

Punktestand oder optimaler Punktestand:
Die Summe aller Elemente einer Zugfolge für N bestimmt die Punktzahl((a(m))(N)).

z.B.
Die Punktzahl ((19,15,14,20,16,12,18)(20)) ist identisch mit 19+15+14+20+16+12+18=114.
Die Punktzahl((19,10,15,20,12,14,18,16)(20)) ist identisch mit 19+10+15+20+12+14+18+16=124.

Im Allgemeinen ist die Punktzahl für eine Zugfolge (a(m))(N)= (n1, n2,...,nm)(N) definiert als Punktzahl((a(m))(N)) = Summe(i=1 bis m)ni.

Für jedes N existiert per Definition eine optimale Punktzahl, die wie folgt definiert ist
optimale Punktzahl = optimale Punktzahl({1,...,N}) = optimale Punktzahl(N) = Maximum((Punktzahl((a(m))(N))), für alle (a(m))(N))
z.B. optimale Punktzahl(20) = 124.

„Optimaler Pfad“ oder „Optimale Zugfolge“:
Wenn für eine Zugfolge ((a(m))(N)) die optimale Punktzahl(N) = Punktzahl((a(m))(N)) ist, so wird sie als optimaler Pfad oder optimale Zugfolge bezeichnet.

2. Theoreme and Beobachtungen

Theorem 1

Die optimale Zugfolge ist nicht immer eindeutig.

Beweis durch Beispiel:
Siehe die verschiedenen Zugfolgen für N=20 oder N=158 im Anhang. q.e.d.

Dan Hoey [2] schreibt:

„The game can be played on an arbitrary vertex-labeled directed graph, and graph-theoretical constraints are useful in reducing the search space. For instance, after several moves are made, the divisibility graph on the remaining choices often becomes disconnected, and the components can be optimized independently of one another.“

„Das Spiel kann auf einem beliebigen gerichteten Graphen mit beschrifteten Knoten gespielt werden. Graphentheoretische Einschränkungen sind nützlich, um den Suchraum zu reduzieren. Zum Beispiel spaltet sich der Teilbarkeitsgraph der verbleibenden Wahlmöglichkeiten oft nach mehreren Zügen. Diese Komponenten des Graphs können dann unabhängig voneinander optimiert werden.“

Mit anderen Worten: Die Reihenfolge der Komponenten kann beliebig sein, was zu mehreren optimalen Zugfolgen führt.

Dan Hoey [2] hat bereits gezeigt, dass die beste Wahl für den ersten Zug immer die höchste Primzahl ist, die N nicht übersteigt. Er gab dafür aber keinen strengen Beweis (nur Plausibilität). Lassen Sie es mich auf eine formale mathematische Weise tun.

Theorem 2

Angenommen, {1,...N} mit N ist Element von |N, ist die gegebene Zahlenmenge. hp ist die höchste Primzahl der Zahlenmenge, d.h. hp = Maximum(i), i ist Element von {1,...N} und hp ist prim. Dann muss der erste Zug, um eine optimale Punktzahl(N) zu erhalten, immer die höchste Primzahl hp sein.

Beispiel:
Sei {1,2,3,4} die Zahlenmenge. hp=3 ist die höchste Primzahl. Für dieses Beispiel kann man leicht sehen, dass die 3 an den Zahlenhai verloren geht, wenn man die 3 nicht im ersten Zug nimmt. Wenn man also mit der 3 im ersten Zug beginnt, können Sie sich diese Zahl für ihr Ergebnis sichern.

Beweis durch Widerspruch
Angenommen, (a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge und n1 als Zahl für den ersten Zug ist keine Primzahl.
Das heißt, optimale Punktzahl(N) = Punktzahl((n1,n2,...,nm)(N)).
Als zusammengesetzte Zahl hat n1 mindestens zwei ganze Teiler. Einer dieser Teiler ist 1.
Somit kann die Punktzahl von ((n1,n2,...,nm)(N)) durch die Zugfolge
((p,n1,n2,...,nm)(N)), mit Punktzahl ((p,n1,n2,...,nm)(N)) = p + Punktzahl((n1,n2,...,nm)(N))) erhöht werden. p ist prim und Element der Zahlenmenge.
Schließlich ist die Punktzahl((hp,n1,n2,...,nm)(N)) > Punktzahl((p,n1,n2,...,nm)(N)) mit hp als höchster Primzahl in der Zahlenmenge.
Dies steht jedoch im Widerspruch zu unserer Annahme. q.e.d.

Dan Hoey [2] berechnete optimale Ergebnisse bis zu N=158. Er erwähnt, dass die optimale Zugfolge im Falle, dass N eine Primzahl ist, einfach identisch mit der optimalen Zugfolge von N-1 ist, wobei die führende Primzahl in der Zugfolge für N-1 durch N ersetzt wird. Dieser Hinweis führt uns zu Theroem 3.

Theorem 3

Angenommen, {1,...N} ist die gegebene Zahlenmenge, wobei N ein Element von |N ist. (a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge. Wenn N+1 eine Primzahl ist, dann ist eine optimale Zugfolge für die Zahlenmenge {1,...N+1} gegeben durch (N+1,n2,...,nm)(N+1).

Beispiel:
Sei {1,2,3,4} die Zahlenmenge. Die optimale Zugfolge ist (3,4)(4). 5=4+1 ist eine Primzahl. In diesem Fall ist die optimale Zugfolge (5,4)(5).

Beweis:
(a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge.
Die optimale Punktzahl ist: Punktzahl(N) = Summe(i=1 bis m)ni = n1 + Summe(i=2 bis m)ni.
Theorem 2 besagt, dass n1 die höchste Primzahl in der Zahlenmenge {1,...,N} ist.
Also nimmt n1 den reellen Teiler 1 aus der Zahlenmenge weg und somit kann die verbleibende Zugfolge (n2,...,nm) als optimale Zugfolge für die Zahlenmenge {2,...,N}\{n1} interpretiert werden.
Theorem 2 besagt, dass der erste Zug für die Zahlenmenge {1,...,N,N+1} die höchste Primzahl ist, also N+1.
Die übrigen Zahlen bilden die Zahlenmenge {2,...,N}.
Auf Seite 34 in [5] wird bewiesen, dass für alle n > 1, n ist Element von |N, mindestens eine Primzahl p existiert, mit n < p < 2n.
Es existiert also eine Primzahl p zwischen floor(N/2) und N.
Außerdem ist floor(N/2)+1 <= p, was bedeutet 2*floor(N/2)+2 <= 2p.
Außerdem kann man leicht sehen, dass N < 2*floor(N/2)+2 ist. Das bedeutet, dass N/2-1 < floor(N/2) ist.
Das bedeutet, dass bei n1 >= p, n1 nicht zur optimalen Punktzahl von {2,...,N} beitragen kann, weil n1 kein Teiler für eine Zahl in der Zahlenmenge ist und n1 nur 1 als Teiler hat, die nicht in der Zahlenmenge ist.
Also ist die optimale Punktzahl({2,...,N}) = optimale Punktzahl({2,...,N}\{n1}).
Somit ist die optimale Punktzahl(N+1) = N+1 + optimale Punktzahl({2,...,N}\{n1}) = Punktzahl((N+1,n2,...,nm)(N+1)). q.e.d.

Theorem 4

Angenommen, {1,...N} ist die gegebene Zahlenmenge, wobei N ein Element von |N ist. (a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge. Dann ist die optimale Punktzahl((n1,n2,...,nm)(N)) <= N(N+1)/2 - Summe(i=1 bis N, i ist prim, i!=hp) i - 1, mit hp als höchster Primzahl.

Beispiel:
Sei {1,...,20} die Zahlenmenge. Nimmt man die größte Primzahl 19, so wird 1 als freier Teiler von 19 vom Zahlenhai genommen. Die Primzahlen 2,3,5,7,11,13,17 befinden sich noch in der Zahlenmenge. Wenn Sie eine von ihnen auswählen, trägt diese Zahl zur Punktzahl des Zahlenhais bei. Aber diejenigen < N/2 (hier also 2,3,5 und 7) könnten noch ein freier Teiler einer anderen zusammengesetzten Zahl sein.

Beweis:
Für die gegebene Zahlenmenge {1,...,N} ist die Summe aller Zahlen nach C.F. Gauß N(N+1)/2.
Nach dem letzten Zug werden alle Zahlen entweder an den Spieler oder an den Zahlenhai verteilt,
also optimale Punktzahl(N) + Punktzahl des Zahlenhais = N(N+1)/2.
Gemäß Theorem 2 muss der erste Zug die höchste Primzahl hp sein, um die optimale Punktzahl zu erreichen.
Also trägt hp zur optimalen Punktzahl bei, 1 als realer Teiler zur Punktzahl des Zahlenhais.
Alle anderen Primzahlen haben nun keine echten Teiler mehr, d.h. sie tragen alle zum Ergebnis des Zahlenhais bei. Wenn Sie eine von ihnen auswählen, wird die Primzahl zur Punktzahl des Zahlenhais hinzugefügt. Ist die Primzahl ein echter Teiler einer anderen Zahl, so trägt auch diese Zahl zur Punktzahl des Zahlenhais bei. Somit tragen alle Primzahlen zur Punktzahl des Zahlenhais bei.
Das Optimum ist also Punktzahl((n1,n2,...,nm)(N)) <= N(N+1)/2 - Summe(i=1 bis N, i ist prim, i!= hp)i - 1. q.e.d.

Dan Hoey [2] erwähnt, dass der zweite Zug normalerweise das höchste Quadrat einer Primzahl ist, die N nicht überschreitet. Wenn man seine optimalen Zugfolgen analysiert, scheint dies in der Regel richtig zu sein. Aber leider ist das nicht in allen Fällen richtig.

Theorem 5

Angenommen, {1,...N} ist die gegebene Zahlenmenge, wobei N ein Element von |N ist. (a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge. Dann ist der zweite Zug n2 nicht immer das höchste Quadrat einer Primzahl, die nicht größer als N ist.

Beweis:
Die folgende Abbildung zeigt einen Teilbarkeitsgraphen ([5], Seite 25) von N=8.
Nach Theorem 2 muss der erste Zug die höchste Primzahl sein, also 7. Nach diesem Zug kann der Graph modifiziert werden. Die 1 als Teiler von 7 und die Primzahlen 5 und 7 können aus dem Graphen entfernt werden. Wenn der zweite Zug das höchste Quadrat einer Primzahl ist, die nicht größer als 8 ist, muss es 4 sein. Wählt man 4, kann der Teiler 2 von 4 aus dem Graphen entfernt werden und es bleiben die 8, 6 und 4 übrig. Nur die 6 hat noch einen freien, reellen Teiler übrig. Daher kann auch die 8 aus dem Graphen entfernt werden. Der letzte Zug ist also die 6 und die Zugfolge ist (7,4,6), was zur Punktzahl 17 führt.
Aber für N=8 ist die optimale Punktzahl 21 und eine optimale Zugfolge ist (7,8,6). q.e.d.

Weitere Beispiele sind die Fälle N=20 und N=120. Die entsprechenden optimalen Zugfolgen sind unten aufgeführt. Im Fall von N=20 ist sogar das höchste Quadrat einer Primzahl, die 20 nicht überschreitet, nämlich 9, nicht Teil der optimalen Zugfolge. Für N=120 ist 25 der zweite Zug anstelle von 49. Für diese Zugfolgen habe ich keine Beweise. Sie beruhen auf den Ergebnissen von Dan Hoey [2].

Beobachtung 1

Angenommen, {1,...N} ist die gegebene Zahlenmenge, wobei N ein Element von |N ist. (a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge. Dann ist die optimale Punktzahl((n1,n2,...,nm)(N)) >= hp + Summe(i=K bis N, i ist nicht prim), wobei hp die höchste Primzahl ist. Die Auswahl von K erfolgt so, dass hp + Summe(i=K bis N, i ist nicht prim) >= N(N+1)/4 und hp + Summe(i=K+1 bis N, i ist nicht prim)i < N(N+1)/4. Oder mit anderen Worten, eine optimale Zugfolge muss mindestens alle Zahlen i enthalten, i ist Element {K,...,N} und i ist nicht prim und hp.

Das Spiel enthält eine Funktion zur Überprüfung der Beobachtung. Das Ergebnis wird auf der Konsole ausgegeben, nachdem eine der unten genannten Strategien ausgeführt wurde.

Beobachtung 2

Angenommen, {1,...N} ist die gegebene Zahlenmenge, wobei N ein Element von |N ist. Für N>3 existiert immer eine Zugfolge (a(m))(N)) = (n1,n2,...,nm)(N) mit score((n1,n2,...,nm)(N)) > N(N+1)/4. Mit anderen Worten: Sie können immer gegen den Zahlenhai gewinnen. Meine Hilfsfunktion erzeugt immer eine solche Zugfolge. Details dazu finden Sie im nächsten Kapitel.

Beobachtung 3

Angenommen, {1,...N+1} ist die gegebene Zahlenmenge, wobei N+1 ein Element von |N ist. (a(m))(N)) = (n1,n2,...,nm)(N) ist eine optimale Zugfolge. Dann ist
a(m))(N+1)) = (N+1,n2,...,nm)(N) ein optimaler Zug, wenn N+1 prim ist, sonst f((n1,n2,...,nm)(N))
mit f:(n1,n2,...,nm)(N)->(n1,n2,...,nm')(N+1) als noch nicht bekannter Algorithmus oder Funktion.
Wenn N+1 eine Primzahl ist, beweist Theorem 3 den ersten Teil des rekursiven Algorithmus.
Meine Ergebnisse für eine Funktion f finden Sie im nächsten Kapitel.

Beobachtung 4

Die Primzahlen spielen eine entscheidende Rolle, um die optimale Punktzahl zu erreichen. Geben Sie dem Zahlenhai die Primzahlen so langsam wie möglich. So langsam wie möglich bedeutet, nur eine oder zwei pro Zug. Während des Spiels wird es passieren, dass zusammengesetzte Zahlen zu Pseudo-Primzahlen werden. Ich nenne zusammengesetzte Zahlen, die keine freien, reellen Teiler mehr haben Pseudo-Primzahlen.
Z.B. hat 12 zu Beginn eines Spiels die realen, freien Teiler 6,4,3,2 und 1. Nach einer Weile Spielzeit hat 12 vielleicht nur noch die freien, realen Teiler 6 und 4. Da 3,2 und 1 weg sind, haben 6 und 4 keine freien, reellen Teiler mehr, so dass 6 und 4 nach meiner Definition Pseudo-Primzahlen sind.
Bei größeren Zahlenmengen müssen auch die Pseudoprimzahlen berücksichtigt werden. Meine vorherige Aussage könnte also dahingehend geändert werden, dass ich versuche, dem Zahlenhai so langsam wie möglich Prim- und Pseudoprimzahlen zu geben.

Beobachtung 5

Für die Länge l der optimalen Zugfolge gilt die folgende Gleichung:
l >= floor(N/2) - (pi(N)-pi(ceil(N/2)) + 1, pi(N) ist die Anzahl der Primzahlen <= N
z.B. N=3, floor(3/2)=1, ceil(3/2)=2, -> l >= 1-(2-1)+1 = 1
z.B. N=10, floor(10/2)=ceil(10/2)=5, -> l >= 5-(4-3)+1 = 5

z.B. N=79, floor(79/2)=39, ceil(79/2)=40, -> l >= 39-(22-12)+1 = 30
z.B. N=100, floor(100/2)=ceil(100/2)=50, -> l >= 50-(25-15)+1 = 41

3. Strategien

Wenn man ein wenig mit dem Spiel herumspielt, wird man irgendwie zu dem Schluss kommen, dass die optimale Zugfolge davon abhängt, die obligatorischen Faktoren so langsam wie möglich zu verbrauchen [2]. Einen guten Überblick über einige Strategien finden Sie in [3]. Ich habe unabhängig davon ähnliche Strategien gefunden. Die interessantesten Strategien aus der Quelle werde ich im nächsten Kapitel erläutern.

Tronos Strategie #3, #7 and #9 (Gewinner-Strategie, auch genutzt für die Hinweise an den Benutzer)

In [3] erwähnt John Trono einige Strategien, um eine Zugfolge zu berechnen, um das Spiel zu gewinnen und eine möglichst hohe Punktzahl zu erreichen. Ich habe versucht, seine Strategie #7 umzusetzen. Mit dieser Strategie hat er die höchsten Punktzahlen für N=256 und N=512 berechnet (Anhang). Seine Strategie ist wie folgt definiert.

Gehen Sie in 3 Schritten vor: Wählen Sie die größte Primzahl; wählen Sie dann die ungeraden Zahlen, beginnend mit der größten und arbeiten Sie abwärts, bis N/2 erreicht ist; und wählen Sie schließlich die geraden Zahlen, beginnend mit N/2+1 und arbeiten Sie aufwärts zu N.

Leider konnte ich seine Ergebnisse nicht reproduzieren. Daher beschloss ich, seine Strategie Nr. 3 umzusetzen. Diese Strategie ist wie folgt definiert:

Nehmen Sie immer die größte aktive ganze Zahl mit nur einem aktiven Teiler (kleiner als dieser).

Diese Strategie funktioniert sehr gut bei größeren Zahlenmengen. Sie können sehr schnelle Zugfolgen mit hoher Punktzahl berechnen.

Eine Abwandlung dieser Strategie besteht darin, nur nach der größten aktiven ganzen Zahl mit den wenigsten Teilern zu suchen. Diese Strategie kann so modifiziert werden, dass man die größte aktive ganze Zahl mit den wenigsten Teilern sucht, die größer als ein bestimmter Grenzwert ist. Tronos Strategie #9 verwendet genau diese Strategie mit dem Grenzwert N/2-1. Tatsächlich war dies auch meine erste Strategie. In meinem Fall ist der Grenzwert N/2+1. Übrigens ist diese Strategie für das Support-Feature implementiert. Mein Ansatz war, eine immer gewinnende Strategie zu finden, d.h. für jede Zahlenmenge eine Zugfolge zu finden, die einen gegen den Zahlenhai gewinnen lässt. Offensichtlich löst diese Strategie das Problem. Für die weitere Diskussion nenne ich diese Strategie die "Winner"-Strategie.

Schus Zufallsstrategie 1

Mein nächster Schritt bestand darin, eine Strategie zu finden, die eine optimale Punktzahl oder zumindest eine Punktzahl, die einer Optimalen so nahe wie möglich kommt, ermitteln kann. Ich fand heraus, dass die Ergebnisse meiner bisherigen "Winner"-Strategie von einem geschickt gewählten Grenzwert abhängen. Werfen wir einen Blick auf ein Beispiel:
Die Größe N der Zahlenmenge ist 20. In diesem Fall würde mein Grenzwert zu 20/2+1 = 11 berechnet werden, also genau 12, denn 11 ist eine Primzahl, die keine Wirkung hat. Somit würde mein Algorithmus nicht die optimale Punktzahl finden, da 10 benötigt wird Anhang). Aber auch Tronos Strategien würden scheitern, denn der Grenzwert wäre 20/2-1=9. Tronos Strategien würden also die 9 als zweiten Zug nehmen, was nicht zu einer optimalen Punktzahl führt. Für N=20 würde der Grenzwert 10 meine Strategie zu einem optimalen Ergebnis führen. Im Gegensatz dazu ist diese Grenze für N=21 oder N=19 nicht optimal. Auch hier wäre der Grenzwert 9 die optimale Wahl.
Die erste Idee, die auf dieser Erkenntnis beruht, könnte also sein, für eine gegebene Zahlenmenge den Algorithmus mit verschiedenen Grenzwerten von 0 bis N/2+1 laufen zu lassen, um eine möglichst hohe Punktzahl zu erhalten. Aber wenn man sich weiter zu größeren Zahlenmengen bewegt, wird man sehen, dass dies nicht ausreicht. Irgendwie sollte sich der Grenzwert auch während des Spiels ändern.
Mein zweiter Ansatz war, während des Spiels nach jedem Zug den Grenzwert zufällig von 0 auf N/2+1 zu ändern. Dies führt bereits zu besseren Ergebnissen. Aber um bessere Ergebnisse für größere Zahlenmengen zu erhalten, müssen Pseudo-Primzahlen berücksichtigt werden. Pseudo-Primzahlen werden scheinbar selten verwendet, aber manchmal werden sie eingesetzt. Pseudo-Primzahlen befinden sich im Intervall N bis N/2+1. Also habe ich eine zweite Grenze eingeführt, die sich ebenfalls zufällig im Intervall N bis N/2+1 ändert. Meine Strategie kann also wie folgt ausgedrückt werden:

Schritt 1: Berechne zufällig einen ersten Grenzwert aus dem Intervall N/2+1 bis hinunter zu 0.
Schritt 2: Nimm die größte aktive ganze Zahl mit den wenigsten Teilern im Intervall N bis hinunter zu einem ersten Grenzwert.
Schritt 3: Berechne zufällig einen zweiten Grenzwert aus dem Intervall N bis hinunter zu N/2+1.
Schritt 4: Nimm die größte Pseudoprimzahl im Intervall N bis hinunter zu einem zweiten Grenzwert.
Schritt 5: Nimm als aktuellen Zug zufällig eine Zahl aus den in Schritt 2 und 4 berechneten Zahlen.
Step 6: Wiederhole ab Schritt 1, solange noch freie Zahlen vorhanden sind.

Schus Zufallsstrategie 2

Diese Strategie ist ähnlich wie Strategie 1. Die Modifikation beruht darauf, dass mindestens die größte ganze Zahl mit den wenigsten Teilern im Intervall von N bis hinunter zu N/2+1 berücksichtigt werden soll. Nur wenn es im Intervall N/2+1 bis hinunter zu 0 Zahlen mit einer geringeren Anzahl von Teilern gibt, sind auch diese von Interesse. Die Strategie 2 kann also wie folgt beschrieben werden:

Schritt 1: Nimm die größte aktive ganze Zahl mit dem kleinsten Teiler aus dem Intervall N bis N/2+1.
Schritt 2: Berechne zufällig einen ersten Grenzwert aus dem Intervall N/2+1 bis hinunter zu 0.
Schritt 3: Nimm alle aktiven ganzen Zahlen im Intervall N bis hinunter zu einem ersten Grenzwert,
dessen Anzahl der Teiler kleiner ist als die Anzahl der Teiler der Zahl in Schritt 1.
Schritt 4: Berechne zufällig einen zweiten Grenzwert aus dem Intervall N bis hinunter zu N/2+1.
Schritt 5: Nimm die größte Pseudoprimzahl im Intervall N bis zu einem zweiten Grenzwert.
Schritt 5: Nimm als aktuellen Zug zufällig eine Zahl aus den in Schritt 1, 2 und 4 berechneten Zahlen.
Schritt 7: Wiederhole ab Schritt 1, solange noch freie Zahlen vorhanden sind.

Schus rekursive Strategie

Auf der Grundlage von Beobachtung 3 habe ich einen rekursiven Algorithmus entwickelt. Um eine geeignete Funktion f zu definieren, analysierte ich alle optimalen Zugfolgen. Ich beobachtete, dass sich beim Übergang von der optimalen Zugfolge von N zu N+1 nur ein Teil der Zahlen ändert. Sehr oft trat einer der drei folgenden Fälle auf:

Fall 1: Es wurde nur die neue Zahl N+1 hinzugefügt und der Rest der Zugfolge von N wird beibehalten.
Fall 2: Die neue Zahl N+1 wurde hinzugefügt und der Rest der Zugfolge N wird bis auf eins beibehalten.
Fall 3: Die neue Zahl wurde hinzugefügt und der Rest der Zugfolge N wird beibehalten, bis auf eine, die durch eine neue Zahl ersetzt wurde.

Generell ist zu beobachten, dass von N -> N+1 entweder die Länge der Zugfolge beibehalten oder um eins erhöht wird. Es wird immer die neue Zahl N+1 beibehalten und der Rest der Zugfolge N wird neben r Zahlen beibehalten, wobei r ein Element von {0,...,R}, R ein Element von |N und R >=7 ist. Ich habe die Unterschiede von N->N+1 unten aufgelistet. Es sind nur Unterschiede mit r > 2 erwähnt. Alle anderen Unterschiede sind 1 oder 2.
