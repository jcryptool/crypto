About SHA-3

The Secure Hash Algorithm is a family of cryptographic hash functions published by the National Institute of Standards and Technology (NIST) as a U.S. Federal Information Processing Standard (FIPS) (see Wikipedia). In 2012, the NIST has chosen the hash function Keccak as SHA-3 and thus successor of the SHA-1/2 family.

JCrypTool implemented three of candidates that achieved the second round of the SHA-3 selection process.

More information about these algorithms is available on the sub sites.

Usage of the dialog

While the dialogs for MD5 and SHA-256 don't need any input and just write their output to the hexeditor within JCT, the dialog for the SHA-3 candidates has more options.

After choosing the algorithm and the wished bitlength in the dialog for the SHA-3 candidates, you can determine whether you want to create a hash value or verify a hash value. In case you create a hash value there are two ways where to place the output. ECHO has the additional option to salt the algorithm (with 32 hex characters).

Generate hash

The hash is generated always out of the content of the active editor.

The output is placed in one the following two mediums:

  • Hexeditor: Displays the hash value in a hex editor.
  • Texteditor: Displays the hash value as hex values in a text editor (blocked in groups of 2).

By clicking on the Finish button the hash value will be generated.

Verify hash

A new hash value is generated from the editor content. This new hash value is compared with a given hash value to be entered in the SHA-3 dialog. Both hash values (the given and the generated one) and also the result of the comparison are shown as output in a new text editor tab.

When selecting the option Verify Hash, a new field is available to enter the given hash value.

By clicking on the Finish button the verification will be started.
