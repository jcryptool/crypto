Sudoku Solver

This plug-in helps to solve three kinds of sudoku, namely the normal sudoku, killer sudoku and hexadecimal sudoku.

This help file consists of four main sections: puzzle explanation, sudoku tactics, plug-in instructions and file handling.

Puzzle Explanation

The normal variant is well known, it consists of a 9x9 grid where in each position a number ranging from 1 to 9 must be entered. But these numbers must be entered in such a way that it is unique in its row, column and 3x3 sub box (indicated by the bold lines).

A hexadecimal sudoku is merely an extension of this puzzle to a 16x16 grid with 4x4 sub boxes. Now, 16 unique numbers must be entered, often denoted by 0 through 9 and A through F, hexadecimal numbers. The restrictions are analogous.

Most people consider the killer sudoku as the most difficult of the three types. In this type, different boxes are defined with mathematical operations (addition, subtraction, multiplication and division). The numbers entered in these boxes must have the correct result when the operation is performed. Again, numbers must be unique in its row, column and "operation box". It is also possible that the number must be unique in its 3x3 sub box as well. The plug-in allows you to turn this 'box rule' on and off.

Sudoku Tactics

Since sudoku has become popular several tactics have been published online. Some of these tactics are implemented in this solver to aid the user in solving a sudoku puzzle. For the normal sudoku this plug-in does not have any built in tactics at this point. This is due to the fact that a normal sudoku can easily be solved by brute force (trying all possibilities).

A killer sudoku is much harder to brute force because of the stranger restrictions. Nonetheless, no tactics are built in at this point. This is due to the fact that no "good" tactics could be found at the moment. The brute-force solver calculates the possibilities per square with respect to the mathematical restrictions and starts to try them in a smart way (least possibilities first).

Hexadecimal sudoku probably is the hardest to brute force because of the increased puzzle size and thus possible numbers. So for this type there are several tactics built in. These tactics are explained for normal sudoku on www.su-doku.net and www.sudokuoftheday.com. The following are available:

  • One possible (not explained, obviously)
  • Naked single
  • Hidden single
  • Block and Column
  • Naked subset
  • Candidate line
  • Double pair
  • Multiple lines

Plug-in Instructions

The plug-in has two modes; Enter and Solve. At startup it will be in Enter mode in which either an existing or standard puzzle can be loaded or a puzzle can be entered manually. Entering a Normal or Hexadecimal sudoku is straightforward; simply enter the given numbers in the correct squares.

Entering a Killer sudoku is a bit more complicated. Given numbers can be entered, but the areas with mathematical operation and results must be added as well. To do so, consecutively click the squares for one area, turning them red. Once all the squares in an area are selected, choose the operator from the menu on the left and enter the required result. Note that only bordering squares can be chosen in one area.

After entering a puzzle switch to Solve mode to start solving. To solve a sudoku, simply click a square and type in the number that should go there. The plug-in prevents you from entering a number that violates one of the rules. If you want to make solving a bit easier, you can let the plug-in show which numbers are possible for each square. If you're stuck; you can ask for a hint. The plug-in will select an empty square at random and fill out the correct number. Note that the Hint-button might not be available immediately when switching to Solve Mode, the plug-in will first solve the puzzle in the background.

The plug-in has the possibility to save and load puzzles so you can stop at any time. Also, you could download puzzles from the Internet and open them in JCT. The file format demands are different for each type of puzzle, see file handling.

The autofill button can be clicked to fill out one square where only one number is possible. The square that is filled out will blink red after this is done.

In case of the hexadecimal puzzle, the strategy buttons apply one of the strategies to the puzzle and fill out the results. The strategy is repeated until no new entries are discovered.

File Handling

The plug-in supports different types of file formats for the different puzzles. Below it is explained what is expected in a file.

Normal Sudoku Example File

The file consists of 9 lines of 9 characters each. The file is constructed like a Soduko. Thus, the value 4 in the second line represents the first sign that the first character in the sudoku in the second line is a 4. A 0 represents a non-existent value.

At the end of each line in the normal sudoku file is a \n.

Explanation of the structure of the killer sudoku file

The upper block consists of 81 characters (9 x 9 fields) and represents the areas in the killer sudoku that belong together. The fields with the same character always belong to an area. For example, the fields with a colon form an area.

From the 10th line of the file, the areas indicated in the upper area are provided with the information whether they are added (+), subtracted (-), multiplied (*) or divided (:). For example, the area Colon defined above is an area in which the specified values in Sudoku must be 11 in total.

At the end of each line in the killer sudoku file is a \n.

Example of a hex sudoku file

The file consists of 256 characters representing the 256 fields in a 16 x 16 sudoku. The possible value range is from 0 to F. A 0 stands for an empty field. A = 10, B = 11, C = 12, D = 13, E = 14 and F = 15.

At the end of each line in the hex sudoku file is a \n.
