2) eXtended-Merkle-Signature-Scheme (XMSS)

Einleitung

Das eXtended-Merkle-Signature-Scheme-Verfahren (XMSS) basiert auf dem Merkle-Signature-Scheme (MSS) und erweitert dieses. Die Hauptunterschiede zum MSS-Verfahren sind der Einsatz einer Bitmaske und eines sogenannten L-Trees. Durch den Einsatz von Bitmasken und des L-Trees kann auf eine kollisionsresistente Hashfunktion verzichtet und durch eine second-preeimage-resistente Hashfunktion ersetzt werden. Der Vorteil hierbei ist, dass eine second-preeimage-resistente Hashfunktion eine schwächere Annahme ist, und somit schwerer zu brechen ist. Ein weiterer Vorteil, der sich aus dem Verzicht einer kollisionsresisten Hashfunktion ergibt ist, dass die erstellte Signatur kleiner ist. XMSS benutzt als One-Time-Signature (OTS) Winternitz-OTS (WOTS). Es können insgesammt 2^h Nachrichten mit einer Länge von m-Bytes signiert werden.

Bitmaske

Die Bitmaske ist ein pseudozufälliger binärer Wert. Durch ihren Einsatz kann auf eine kollisionsresistente Hashfunktion verzichtet werden. Bitmasken werden im XMSS-Tree und im L-Tree verwendet.

L-Tree

Bei den L-Trees werden dieselben Hashfunktionen wie beim XMSS-Baum verwendet, jedoch eine andere Bitmaske. Die verwendete Bitmaske ist wiederum bei jedem L-Tree dieselbe.

3) Bedienung

Tab: Seed- und Schlüsselerzeugung

Auf dieser Seite kann zwischen den einzelnen Verfahren gewechselt werden. Des Weiteren kann hier der Seed festgelegt werden, die Anzahl der Secret-Keys definiert und die Secret-Keys erzeugt werden.

Hier muss der Seed festgelegt werden, dies wird benutzt um die Secret-Keys zu generieren. Es kann entweder ein eigener numerischer Seed benutzt werden, oder mit dem Button Generate new Seed ein zufälliger Seed erzeugt werden.

Wenn als Verfahren eXtended-Merkle-Signature-Scheme (XMSS) gewählt wurde, so erscheint zusätzlich noch ein Feld in dem man den Seed für die Bitmaske definieren kann.

Tab: MerkleTree

In diesem Tab kann der Merklebaum inspiziert werden. Der Baum besteht aus den Leafs (Unterste Ebene), den Nodes und dem Root-Node, auch Merkle-Public-Key genannt. Es kann auf die einzelnen Nodes geklickt werden, dann wird in Gelb der Pfad zur Wurzel angezeigt. In Rot wird der Verifizierungspfad angezeigt, welcher ein Teil der Signatur ist. Des Weiteren wird, wenn man einen Node beziehungsweise Leaf anklickt, der Hash des ausgewählten Node angezeigt.

Tab: Signierung

In diesem Tab können Signaturen generiert werden. Diese werden mit den in Tab Eins generierten Keys erzeugt. Hier kann ein beliebiger Text eingegeben werden, welcher nach dem Drücken auf Sign Message signiert wird

Die Signatur befindet sich im hier abgebildeten Fenster. Auch angegeben ist der Index der Signatur und Länge in Bytes.

Tab: Verifikation

Im Tab Verifikation kann die zuvor erstellte Signatur verifiziert werden. Das korrekte Leaf ist hierbei schon vorab ausgewählt. Wenn man auf Verify klickt, so wird der ausgewählte Leaf verifiziert.

Wenn man ein anderes Leaf auswählt und auf verifizieren klickt, so schlägt die Verfikation fehl.
